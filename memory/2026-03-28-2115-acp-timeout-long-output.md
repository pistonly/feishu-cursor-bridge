## 背景

在验证飞书卡片承载极长正文时，需要区分：**飞书 patch 竞态** 与 **ACP/会话生命周期超时**。

## 关键结论

- 约 **4000 字**量级：压测未观察到 ACP 侧异常中断。
- 约 **20000 字**（与 `reference/feishu-lark-md-stress-20k.md` 同级）：在**最大压测**下，若单次流程 **超过约 10 分钟**，会出现 **ACP 适配器中断**（会话/链路被掐断）。耗时长本身易触发超时，此时不应先归因于飞书 patch 串行化是否成功。
- **对照**：同类超长场景下 **Cursor CLI 仍能正常输出**；问题集中在 **ACP 路径超时或适配器生命周期**，而非模型/CLI 不产出。
